# Projects

Projects group work for reporting and **cost roll-up**. The SDK exposes **`v_projects`** for reads and **`client.projects.create`**, **`update`**, and **`delete`** (via public RPCs; permission **`project.manage`**, usually on **admin**). {{ className: 'lead' }}

## Overview

Set **tenant context** before calling these methods. Rows include **`id`**, **`tenant_id`**, **`name`**, optional **`code`** (unique per tenant when set), **`description`**, **`created_at`**, and **`updated_at`**.

## List projects

<CodeGroup title="client.projects.list()">

```ts
const projects = await client.projects.list()
// Returns ProjectRow[]
```

</CodeGroup>

## Get by ID

<CodeGroup title="client.projects.getById(id)">

```ts
const project = await client.projects.getById('uuid-of-project')
// Returns ProjectRow | null
```

</CodeGroup>

## Create project

Creates a project and returns the new **UUID**. Requires **`project.manage`**. **`name`** is required; **`code`** and **`description`** are optional.

<CodeGroup title="client.projects.create(params)">

```ts
await client.setTenant(tenantId)

const projectId = await client.projects.create({
  tenantId,
  name: 'Building A retrofit',
  code: 'RETRO-A', // optional; must be unique per tenant if set
  description: 'HVAC and controls',
})
// Returns string (project UUID)
```

</CodeGroup>

## Update project

Partial updates: pass **`null`** for fields you do not want to change (per RPC contract). Requires **`project.manage`**.

<CodeGroup title="client.projects.update(params)">

```ts
await client.projects.update({
  tenantId,
  projectId,
  name: 'Building A retrofit — phase 2',
  code: null,
  description: null,
})
```

</CodeGroup>

## Delete project

Removes the project row. Work orders that referenced it keep **`project_id`** cleared (FK behavior), not deleted. Requires **`project.manage`**.

<CodeGroup title="client.projects.delete(tenantId, projectId)">

```ts
await client.projects.delete(tenantId, projectId)
```

</CodeGroup>

## Work orders and cost roll-up

Pass **`projectId`** when creating a work order so labor, parts, and vendor costs roll up under that project. Use **`client.costs.costRollup()`** with `groupBy: 'project'` for aggregates.

<CodeGroup title="Work order linked to a project">

```ts
await client.workOrders.create({
  tenantId: 'uuid-of-tenant',
  title: 'Overhaul pump P-101',
  projectId: 'uuid-of-project',
  // ...priority, status, etc.
})
```

</CodeGroup>
