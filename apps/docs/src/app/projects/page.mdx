# Projects

Projects group work for reporting and **cost roll-up**. The SDK exposes `v_projects` for reads and **public RPCs** for create, update, and delete (permission `project.manage`). {{ className: 'lead' }}

## Overview

Set **tenant context** before calling these methods. Rows include `id`, `tenant_id`, `name`, `code`, `description`, `created_at`, and `updated_at`.

## List projects

<CodeGroup title="client.projects.list()">

```ts
const projects = await client.projects.list()
// Returns ProjectRow[]
```

</CodeGroup>

## Get by ID

<CodeGroup title="client.projects.getById(id)">

```ts
const project = await client.projects.getById('uuid-of-project')
// Returns ProjectRow | null
```

</CodeGroup>

## Create, update, delete

Requires **`project.manage`** (typically granted to the tenant **admin** role). Optional `code` must be unique per tenant when set.

<CodeGroup title="client.projects.create / update / delete">

```ts
await client.setTenant(tenantId)

const projectId = await client.projects.create({
  tenantId,
  name: 'Building A retrofit',
  code: 'RETRO-A',
  description: 'HVAC and controls',
})

await client.projects.update({
  tenantId,
  projectId,
  name: 'Building A retrofit — phase 2',
  code: null, // omit from update by passing null where supported
  description: null,
})

await client.projects.delete(tenantId, projectId)
// Work orders with this project_id are left in place; FK sets project_id to null on delete.
```

</CodeGroup>

## Work orders and cost roll-up

Pass **`projectId`** when creating a work order so labor, parts, and vendor costs roll up under that project. Use **`client.costs.costRollup()`** with `groupBy: 'project'` for aggregates.

<CodeGroup title="Work order linked to a project">

```ts
await client.workOrders.create({
  tenantId: 'uuid-of-tenant',
  title: 'Overhaul pump P-101',
  projectId: 'uuid-of-project',
  // ...priority, status, etc.
})
```

</CodeGroup>
