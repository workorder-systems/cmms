# Quickstart

Get the SDK running and make your first API calls in a few minutes. {{ className: 'lead' }}

<Note>
  You need a Supabase project URL and anon key from the [Supabase dashboard](https://supabase.com/dashboard) (Project Settings → API). For tenant-scoped data, sign in with Supabase Auth and set tenant context.
</Note>

## 1. Install the SDK

Install the SDK and its peer dependency:

<CodeGroup>

```bash
npm install @workorder-systems/db @supabase/supabase-js
```

</CodeGroup>

The SDK wraps the Supabase client and exposes domain resources: tenants, work orders, assets, locations, departments, meters, and tenant context helpers.

## 2. Create a client and make your first request

Create a client with your Supabase URL and anon key. You can list tenants (no tenant context needed). For work orders and other tenant-scoped data, sign in and set tenant context first.

<CodeGroup title="Create client and list tenants">

```ts
import { createDbClient } from '@workorder-systems/db'

const client = createDbClient(
  process.env.SUPABASE_URL!,
  process.env.SUPABASE_ANON_KEY!
)

// List tenants the current user is a member of (requires auth)
const tenants = await client.tenants.list()
```

</CodeGroup>

For tenant-scoped resources (work orders, assets, and so on), sign in and set tenant context:

<CodeGroup title="Sign in and list work orders">

```ts
// Sign in (e.g. in your app after login)
const { data } = await client.supabase.auth.signInWithPassword({
  email: 'user@example.com',
  password: 'password',
})

// Set tenant context, then refresh session so JWT carries tenant_id
await client.setTenant(tenantId)
// In a real app you would refresh the session here (getSession → setSession)

const workOrders = await client.workOrders.list()
```

</CodeGroup>
