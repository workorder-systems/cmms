# Labor (technicians & crews)

Technicians, crews, skills, certifications, availability, shifts, work order assignments, labor actuals, and capacity views. Set tenant context before calling these methods. {{ className: 'lead' }}

## Overview

The `client.labor` resource exposes tenant-scoped views, **public write RPCs** for crews, technicians, and crew membership (permissions `labor.crew.manage` and `labor.technician.manage`), and scheduling RPCs:

- **Technicians** and **crews** – workforce and team structure (reads + writes via RPCs)
- **Skills** and **certifications** – tenant catalogs and technician linkage
- **Availability** – recurring patterns and date overrides
- **Shifts** and **shift templates** – scheduled and on-call slots
- **Work order assignments** – which technicians are assigned to which work orders
- **Labor actuals** – aggregated time and cost per work order
- **Technician capacity** – scheduled minutes per technician per day
- **RPCs** – create/update/delete crews and technicians, add/remove crew members; check shift conflicts; generate shifts from templates

## Crews and technicians (writes)

Requires **`labor.crew.manage`** for crews and membership, and **`labor.technician.manage`** for technician records. **`rpc_create_technician`** only accepts `userId` values that already have a **tenant membership** in that tenant.

<CodeGroup title="Crews, technicians, crew members">

```ts
await client.setTenant(tenantId)

const crewId = await client.labor.createCrew({
  tenantId,
  name: 'Day shift — mechanical',
  description: null,
})

await client.labor.updateCrew({
  tenantId,
  crewId,
  name: 'Day shift — mechanical (east)',
  description: 'Covers buildings 1–3',
  leadTechnicianId: 'uuid-of-technician',
  clearLeadTechnician: false,
})

const technicianId = await client.labor.createTechnician({
  tenantId,
  userId: 'uuid-of-auth-user', // must be a member of the tenant
  employeeNumber: 'E-1042',
  defaultCrewId: crewId,
  departmentId: null,
})

await client.labor.updateTechnician({
  tenantId,
  technicianId,
  employeeNumber: null,
  defaultCrewId: null,
  departmentId: null,
  isActive: true,
  clearDefaultCrew: false,
  clearDepartment: false,
})

await client.labor.addCrewMember({
  tenantId,
  crewId,
  technicianId,
  role: 'technician',
})

await client.labor.removeCrewMember(tenantId, crewId, technicianId)
// Sets left_at on the active membership row.

await client.labor.deleteCrew(tenantId, crewId)
```

</CodeGroup>

## Shift conflict detection

Check for overlapping shifts for a technician before creating or updating a shift. Pass `excludeShiftId` when updating so the current shift is not counted as a conflict.

<CodeGroup title="client.labor.checkShiftConflicts(params)">

```ts
const conflicts = await client.labor.checkShiftConflicts({
  technicianId: 'uuid-of-technician',
  startAt: '2026-03-15T08:00:00Z',
  endAt: '2026-03-15T17:00:00Z',
  excludeShiftId: null, // or shift id when updating
})
// Returns ShiftConflictRow[] (id, start_at, end_at, shift_type, label)
```

</CodeGroup>

## Generate shifts from templates

Generate shift rows from shift templates for a date range. Requires tenant membership. Returns the created shifts.

<CodeGroup title="client.labor.generateShiftsFromTemplates(params)">

```ts
const created = await client.labor.generateShiftsFromTemplates({
  tenantId: 'uuid-of-tenant',
  startDate: '2026-03-01',
  endDate: '2026-03-31',
})
// Returns GeneratedShiftRow[]
```

</CodeGroup>

<Properties>
  <Property name="tenantId">
    Required. Tenant UUID.
  </Property>
  <Property name="startDate">
    Required. Start date (YYYY-MM-DD).
  </Property>
  <Property name="endDate">
    Required. End date (YYYY-MM-DD). Must be ≥ startDate.
  </Property>
</Properties>
