# Similar Past Fixes

Search for semantically similar completed work orders. Search by **work order ID** uses RPC only (stored embeddings; no Edge Function). Search by **query text** uses the Edge Function to embed the query, then RPC for similarity search.

## Overview

Similar Past Fixes helps you find past work orders that are semantically similar to a new issue or an existing work order. Use it when creating or editing work orders to suggest relevant past fixes.

- **Indexing** is done server-side by a cron backfill (not by users). Only completed work orders are indexed.
- **Search by work order ID** is via the SDK and uses RPC only (`rpc_similar_past_work_orders_by_work_order_id`). No Edge Function or OpenAI; uses the stored embedding for that work order. If the work order has no embedding, you get no results.
- **Search by query text** is via the SDK and uses the `similar-past-fixes` Edge Function to embed the query, then RPC. Requires tenant context and an authenticated user.

<CodeGroup title="client.similarPastFixes.search(params)">

```ts
await client.setTenant(tenantId)

const results = await client.similarPastFixes.search({
  queryText: 'Motor overload, check connections',
  limit: 5,
  minSimilarity: 0.5,
})
// Returns SimilarPastFixResult[]
```

</CodeGroup>

## Search by query text

Pass free text describing the issue (e.g. title + description). The Edge Function embeds it and returns similar completed work orders.

<CodeGroup title="Search by query text">

```ts
await client.setTenant(tenantId)

const results = await client.similarPastFixes.search({
  queryText: 'Pump not starting, check power supply',
  limit: 5,
})
// Returns SimilarPastFixResult[]
```

</CodeGroup>

## Search by work order ID

Pass an existing work order ID to find similar completed work orders. The SDK calls an RPC that uses the **stored embedding** for that work order (no Edge Function, no OpenAI). Useful when editing an existing work order; the current work order is excluded from results. If the work order has not been embedded yet (e.g. not yet backfilled), the call returns no rows.

<CodeGroup title="Search by work order ID">

```ts
await client.setTenant(tenantId)

const results = await client.similarPastFixes.search({
  workOrderId: 'uuid-of-work-order',
  limit: 5,
  minSimilarity: 0.6,
})
// Returns SimilarPastFixResult[]
```

</CodeGroup>

## Parameters

<Properties>
  <Property name="queryText">
    Free text describing the issue. Mutually exclusive with <code>workOrderId</code>.
  </Property>
  <Property name="workOrderId">
    Work order UUID to embed and search from. Mutually exclusive with <code>queryText</code>.
  </Property>
  <Property name="limit">
    Max results to return (1–50, default 5).
  </Property>
  <Property name="minSimilarity">
    Minimum similarity threshold in [0,1] (default 0.5). Higher = stricter matches; lower = more recall.
  </Property>
</Properties>

## Result shape

Each result is a `SimilarPastFixResult`:

<Properties>
  <Property name="workOrderId">
    UUID of the similar work order.
  </Property>
  <Property name="title">
    Work order title.
  </Property>
  <Property name="description">
    Work order description (optional).
  </Property>
  <Property name="status">
    Status key.
  </Property>
  <Property name="completedAt">
    ISO date string when completed (optional).
  </Property>
  <Property name="similarityScore">
    Similarity score (0–1, higher = more similar).
  </Property>
  <Property name="assetId">
    Optional asset UUID.
  </Property>
  <Property name="locationId">
    Optional location UUID.
  </Property>
  <Property name="cause">
    Optional root cause text for the similar work order.
  </Property>
  <Property name="resolution">
    Optional resolution text describing what was done to fix the issue.
  </Property>
</Properties>
