# Installation

Install the `@workorder-systems/db` SDK to get a type-safe client for Supabase views and RPCs. {{ className: 'lead' }}

## Install the package

<CodeGroup title="Install">

```bash
npm install @workorder-systems/db @supabase/supabase-js
```

</CodeGroup>

Package name: `@workorder-systems/db`. It ships a pre-built `dist/` with TypeScript declarations—no build step needed in your app.

<Note>
  In the monorepo you may see `@workorder-systems/sdk` as the workspace name; use `@workorder-systems/db` when installing from npm.
</Note>

## Peer dependency

`@supabase/supabase-js` is a **peer dependency**. You need to install it yourself (same version range as the SDK’s peer: `^2.39.0`). The SDK uses it under the hood and re-exports types from your `Database` type when used in the same repo.

## Create a client

Import `createDbClient` and pass your Supabase project URL and anon key:

```ts
import { createDbClient } from '@workorder-systems/db'

const client = createDbClient(
  process.env.SUPABASE_URL!,
  process.env.SUPABASE_ANON_KEY!
)

// client.supabase — typed Supabase client
// client.setTenant(tenantId) / client.clearTenant()
// Domain resources:
//   client.tenants, client.workOrders, client.assets, client.locations, client.departments, client.meters
//   client.plugins, client.authorization, client.catalogs, client.pm, client.dashboard, client.audit
```

Works in the browser and in Node.js with no extra config. For edge runtimes (Cloudflare Workers, Vercel Edge), pass a custom `fetch` and optionally turn off session persistence:

```ts
const client = createDbClient(url, anonKey, {
  global: { fetch },
  auth: { persistSession: false },
})
```

## Edge runtimes

In edge environments, pass `global: { fetch }` so the client uses the runtime’s `fetch`. Set `auth: { persistSession: false }` if you don’t need session persistence. For custom storage, use the `auth.storage` option (Supabase `SessionStorage`-compatible).
