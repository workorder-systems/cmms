# Assets

Lists and gets assets for the current tenant; create, update, delete. Set tenant context before calling these methods. {{ className: 'lead' }}

<Note>
  **Downtime / availability:** use **`client.assets.recordDowntime()`** for structured outage events — see [Asset downtime](/asset-downtime).
</Note>

<Note>
  **Barcodes / QR payloads:** optional **`barcode`** on create/update and **`client.assets.resolveByScanCode()`** — see [Floor ops and scanning](/floor-ops-scanning).
</Note>

## List assets

List all assets for the current tenant (from `v_assets`).

<CodeGroup title="client.assets.list()">

```ts
const assets = await client.assets.list()
// Returns AssetRow[]
```

</CodeGroup>

## Get by ID

Fetch a single asset by ID.

<CodeGroup title="client.assets.getById(id)">

```ts
const asset = await client.assets.getById('uuid-of-asset')
// Returns AssetRow | null
```

</CodeGroup>

## Lifecycle fields

`AssetRow` includes lifecycle and planning fields (from `v_assets`). Use them for reporting and replacement planning; create/update asset RPCs may not yet accept all of these (set via database or future API):

- **commissioned_at** – when the asset was placed into service
- **end_of_life_estimate** – estimated EOL date for replacement planning
- **decommissioned_at** – when the asset was decommissioned
- **replaced_by_asset_id** – asset that replaced this one (this asset retired)
- **replacement_of_asset_id** – asset that this one replaced
- **warranty_expires_at** – warranty expiration date
- **service_contract_expires_at** – service/maintenance contract expiration
- **planned_replacement_date** – planned replacement date for capital planning

For alerts (e.g. warranty or EOL approaching), use `client.costs.listLifecycleAlerts()` or `client.costs.assetLifecycleAlerts()`. See [Costs & lifecycle](/costs).

## Warranties (multi-row)

**`warranty_expires_at`** above is a **rollup** from active **`app.asset_warranties`** rows. Use **`v_asset_warranties`** for full history and multiple policies.

<CodeGroup title="client.assets.listWarranties / upsertWarranty">

```ts
const rows = await client.assets.listWarranties(assetId)

const warrantyId = await client.assets.upsertWarranty({
  tenantId,
  assetId,
  expiresOn: '2028-12-31',
  warrantyType: 'standard',
  coverageSummary: 'OEM coverage',
})
```

</CodeGroup>

More context: [Capability inventory — Warranty](/capability-inventory#warranty).

## Barcode and scan resolution

Optional **`barcode`** (unique per tenant when set) is separate from **`assetNumber`**. After a scan, call **`client.assets.resolveByScanCode(tenantId, text)`** (or **`rpc_resolve_asset_by_scan_code`**) to get an asset id: matches **barcode** first, then **asset_number**. See [Floor ops and scanning](/floor-ops-scanning).

## Create asset

Create a new asset. Returns the new asset UUID.

<CodeGroup title="client.assets.create(params)">

```ts
const assetId = await client.assets.create({
  tenantId: 'uuid-of-tenant',
  name: 'HVAC Unit 3B',
  description: 'Main floor HVAC',
  assetNumber: 'AST-003',
  barcode: '4006381333931', // optional; unique per tenant when set
  locationId: 'uuid-of-location',
  departmentId: 'uuid-of-department',
  status: 'active',
})
// Returns string (UUID)
```

</CodeGroup>

<Properties>
  <Property name="tenantId">
    Required. Tenant UUID.
  </Property>
  <Property name="name">
    Required. Asset display name.
  </Property>
  <Property name="description">
    Optional. Description.
  </Property>
  <Property name="assetNumber">
    Optional. Asset number or code.
  </Property>
  <Property name="barcode">
    Optional. Scannable id (unique per tenant when set). See [Floor ops and scanning](/floor-ops-scanning).
  </Property>
  <Property name="locationId">
    Optional. Location UUID.
  </Property>
  <Property name="departmentId">
    Optional. Department UUID.
  </Property>
  <Property name="status">
    Optional. Default `active`. Status key.
  </Property>
</Properties>

## Update asset

Update an existing asset. All fields except `tenantId` and `assetId` are optional.

<CodeGroup title="client.assets.update(params)">

```ts
await client.assets.update({
  tenantId: 'uuid-of-tenant',
  assetId: 'uuid-of-asset',
  name: 'HVAC Unit 3B (updated)',
  description: null,
  assetNumber: null,
  locationId: null,
  departmentId: null,
  status: 'inactive',
})
```

</CodeGroup>

## Delete asset

Delete an asset by tenant and asset ID.

<CodeGroup title="client.assets.delete(tenantId, assetId)">

```ts
await client.assets.delete('uuid-of-tenant', 'uuid-of-asset')
```

</CodeGroup>
