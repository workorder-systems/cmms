# Quickstart

This guide gets you set up with the Work Order Systems TypeScript SDK (`@workorder-systems/db`), creates a client, and shows how to make your first requests. {{ className: 'lead' }}

<Note>
  You need a Supabase project URL and anon key. Get them from the [Supabase dashboard](https://supabase.com/dashboard) under Project Settings → API. For tenant-scoped operations you will also sign in with Supabase Auth and set tenant context.
</Note>

## Choose your client

Install the SDK and its peer dependency:

<CodeGroup>

```bash
npm install @workorder-systems/db @supabase/supabase-js
```

</CodeGroup>

The SDK is a type-safe wrapper over the Supabase client. It exposes domain resources (tenants, work orders, assets, locations, departments, meters) and tenant context helpers.

## Making your first request

Create a client with your Supabase URL and anon key, then call the API. List tenants (no tenant context needed) or, after signing in and setting tenant context, list work orders.

<CodeGroup title="Create client and list tenants">

```ts
import { createDbClient } from '@workorder-systems/db'

const client = createDbClient(
  process.env.SUPABASE_URL!,
  process.env.SUPABASE_ANON_KEY!
)

// List tenants the current user is a member of (requires auth)
const tenants = await client.tenants.list()
```

</CodeGroup>

To get data from tenant-scoped resources (work orders, assets, etc.), sign in with Supabase Auth and set tenant context before calling them:

<CodeGroup title="Sign in and list work orders">

```ts
// Sign in (e.g. in your app after login)
const { data } = await client.supabase.auth.signInWithPassword({
  email: 'user@example.com',
  password: 'password',
})

// Set tenant context, then refresh session so JWT carries tenant_id
await client.setTenant(tenantId)
// In a real app you would refresh the session here (getSession → setSession)

const workOrders = await client.workOrders.list()
```

</CodeGroup>
