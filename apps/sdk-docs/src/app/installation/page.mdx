# Installation

Single TypeScript SDK: `@workorder-systems/db`. Type-safe client over Supabase views and RPCs. {{ className: 'lead' }}

## Package

<CodeGroup title="Install">

```bash
npm install @workorder-systems/db @supabase/supabase-js
```

</CodeGroup>

- **Package name:** `@workorder-systems/db`
- **Build:** The package ships a built `dist/` (TypeScript declarations included). No build step required in your app for consumption; the SDK itself is built with tsup.

<Note>
  In the monorepo, the package may be referenced as `@db/sdk` via workspace; use `@workorder-systems/db` when installing from npm.
</Note>

## Peer dependency

`@supabase/supabase-js` is a **peer dependency**. You must install it yourself (same version range as the SDK’s peer: `^2.39.0`). The SDK uses the Supabase client under the hood and re-exports types from your generated `Database` type when used from the same repo.

## Usage

Import `createDbClient` and create a client with your Supabase project URL and anon key:

```ts
import { createDbClient } from '@workorder-systems/db'

const client = createDbClient(
  process.env.SUPABASE_URL!,
  process.env.SUPABASE_ANON_KEY!
)

// client.supabase — typed Supabase client
// client.setTenant(tenantId) / client.clearTenant()
// client.tenants, client.workOrders, client.assets, client.locations, client.departments, client.meters
```

Works in **browser** and **Node.js** without extra config. For **edge runtimes** (e.g. Cloudflare Workers, Vercel Edge), pass a custom `fetch` and optionally disable or customize session persistence:

```ts
const client = createDbClient(url, anonKey, {
  global: { fetch },
  auth: { persistSession: false },
})
```

## Edge runtimes

In edge environments, pass `global: { fetch }` so the client uses the runtime’s fetch. If you don’t need session persistence, set `auth: { persistSession: false }`. For custom storage, use the `auth.storage` option compatible with Supabase’s `SessionStorage` type.
