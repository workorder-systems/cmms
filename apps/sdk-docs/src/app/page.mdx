# API Documentation

The Work Order Systems API is backed by Supabase: reads go through public views, writes through RPCs. Use the `@workorder-systems/db` TypeScript client for a type-safe, domain-focused API over tenants, work orders, assets, locations, departments, and meters. {{ className: 'lead' }}

## Getting started {{ anchor: false }}

Get your Supabase project URL and anon key from the [Supabase dashboard](https://supabase.com/dashboard) (Project Settings → API). Install `@workorder-systems/db` and create a client with `createDbClient(url, anonKey)`. Sign in with Supabase Auth so list and RPC calls run as an authenticated user; set tenant context before tenant-scoped operations. {{ className: 'lead' }}
